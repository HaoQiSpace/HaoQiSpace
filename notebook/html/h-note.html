    # DTD 文档声明

    现如今是 H5 的时代，故该 DTD 文档声明规范自然是基于 H5 的

    作用：告诉浏览器用哪个版本的 HTML 规范来正确的编译、解析和渲染网页

    <-- DTD 文档声明格式 -->
<!DOCTYPE html>
<-- 基本标准的网页结构 -->
<html>
	<head>
    	<meta charset="UTF-8" />
    	<title>网页标题</title>
    </head>
    <body>
    	网页内容
    </body>
</html>

    # 三大标签

    专业术语：CSS 显示模式

    模式转换：display：block 块级 || inline 行内 || inline-block 行内块级

    块级标签： 独占一行，可设置宽高等属性

    常见：h1 ~ h6，p，div，ul，ol， li 等

    特点：

    • 总是从新行开始
    • 高度、行高、外边距以及内边距等都是可以控制
    • 宽度默认是容器的 100%
    • 可以容纳内联元素和其它块级元素

    行内标签： 不占独立区域，不可设置宽高等属性

    常见：a，span，del，em，strong，ins 等

    特点：

    • 和相邻行内元素在一行上
    • 宽、高无效，但水平方向的 padding 和 margin 可以设置，垂直方向的无效
    • 默认宽度就是它本身内容的宽度
    • 行内元素只能容纳文本或其它行内元素

    行内块标签： 可设置宽高等

    常见：img，input，td 等

    特点：

    • 和相邻行内（行内块）元素在一行上，但是之间会有空白缝隙

    • 默认宽度就是它本身内容的宽度

    • 高度、行高、外边距以及内边距等都是可以控制

    # 假链接

    概念：点击之后不会跳转的链接

    意义：开发初期，未开发出的页面均用假链接代替，开发完成后替换为真链接

    <a href="#">点我：我可以回到顶部</a>
<a href="javascript:">点我：无反应，也称 "死链接"</a>
    1
    2

    区别：如代码描述，故很多回到顶部的功能就是利用 # 做的

    # 锚点

    a 标签可以指定链接跳转，也可以指定某个节点跳转，这个节点称为锚点

    当页锚点

    <-- 1、锁定锚点，格式：# + 锚点 -->
<a href="#center">跳到中部</a>

<-- 2、指定锚点，格式：id + 锚点 -->
<h2 id="center">我是中部</h2>

    跨页锚点

    <-- 1、锁定锚点，格式：路径/xxx.html# + 锚点 -->
<a href="路径/xxx.html#bottom" target="_blank">跳转到 xxx 页面的锚点</a>

<-- 2、对应页面指定锚点，格式：id + 锚点 -->
<h2 id="bottom">我是 xxx 界面的锚点</h2>

    # 列表标签

    作用：给一堆数据添加列表语义，告诉搜索引擎和浏览器这一堆数据是一个整体

    分类：无序、有序和定义列表

    形式：W3school 列表 (opens new window)

    1. ul 标签是用来给一堆数据添加列表语义的，而不是用来给 li 添加小圆点的
    2. ul / ol / dl 标签和 li / dt / dd 标签是一个整体，组合使用
    3. ul / ol / dl 标签中不推荐包含其它标签，其它标签放于 li / dt / dd 标签内
    4. 一个 dt 可以对应一个 dd 或多个 dd，推荐是 一对一 关系

    # 表格标签

    作用：用来给一堆数据添加表格语义

    形式：W3school 表格 (opens new window)

    表格操作：合并单元格

    • 跨行合并：rowspan
    • 跨列合并：colspan
    • 合并思想：将多个内容合并的时候，就会有多于的东西，把它删掉
    • 合并公式：删除的个数 = 合并的个数 – 1
    • 合并顺序：先上 后下 先左 后右

    不用于做布局，只用于做数据展示

    # 表单标签

    作用：收集信息

    形式：W3school 表单 (opens new window)
