    # CSS3 知识梳理

    # 三大特性

    层叠性： 样式冲突，以最后书写样式为准

    继承性： 子标签继承父标签的部分样式

    • 特点：color 和 text-，font-，line- 开头的均可继承
    • 理解：子承父业
    • 优点：简化代码，降低 css 样式的复杂性
    • 口诀：龙生龙，凤生凤，老鼠的儿子生来会打洞

    优先级：

    • !important 声明最高
    • 行内样式 > 内部样式 > 外部样式
    • ID选择器 > class选择器 > 标签选择器 > 通配符
    /**
 * 原则：权值相同，就近原则；不同，高者优先
 * 权值规则
 *   1、统计不同选择器的个数
 *   2、每类选择器的个数乘以相应权值
 *   3、把所有的值相加得出选择器的权值
 **/
#main div.warning h2 {…}
/**
 * Id:1		    class:1		标签:2
 * 1*100=100	1*10=10		2*1=2
 * 权值：100 + 10 + 2 = 112
 **/

    # 样式引用

    优先级：行内 > 内部 > 导入（链入）外部样式：就近原则

    内部样式（嵌入样式）： 写在<head></head>标签内

    <style type="text/css">内部样式</style>
    1

    行内样式： 在开始标签内添加 style 样式属性

    <p style="color:red;">内容</p>
    1

    外部样式： 把 css 样式代码写在独立的一个文件中，再将这个 xx.css 文件引入 html 中

    <link href="xx.css" rel="stylesheet" type="text/css"/>
    1

    上述三种样式引入的方式加载均与 html 文件 同时 加载

    导入式： @import

    <style type="text/css">
	@import url("xx.css");	/* 位于 style 中第一行 */
</style>

    3

    样式加载：html 文件加载完，再加载样式

    # 盒模型

    盒模型相关概念：W3school 盒模型 (opens new window)

    盒模型种类

    CSS 3 中为了优化盒模型，解决宽度偏差带来的错位问题，可以通过 box-sizing 来指定盒模型

    方式：通过指定 content-box、border-box 来改变盒子大小的计算方式

    • W3C 标准盒模型：content-box，width 和 height = 内容的宽高
    • CSS3 盒模型：border-box，width 和 height = 内容的宽度 + border 和 padding

    1. CSS3 盒模型中，如果添加了 padding 和 border，那么真正放置内容的区域会减小
    2. 标准盒模型，增加了像素的计算负担，降低了开发效率
    3. C3 盒模型，页面结构更加稳固

    盒模型居中技巧

    技巧一：外边距实现盒子水平居中

    /**
 * 满足两个条件:
 * 1、必须是块级元素
 * 2、盒子必须指定了宽度
 * 注意：文字居中，text-align: center，其中 0 为像素，可根据需求指定多少
 **/
.header { width: 960px; margin: 0 auto }

    # 浮动

    概念：指设置了浮动属性的元素会脱离标准流的控制，移动到其父元素中指定位置的过程

    目的：为了让多个块级元素同一行显示

    方式：选择器 { float: left | right | none; }

    排序规则

    1. 相同方向上的浮动元素，先浮动的元素会显示在前面，后浮动的元素会显示在后面
    2. 不同方向上的浮动元素，左浮动会找左浮动，右浮动会找右浮动
    3. 浮动元素浮动之后的位置，由浮动元素浮动之前在标准流中的位置来确定

    贴靠现象

    1. 如果父元素的宽度能够显示所有浮动元素，那么浮动的元素会并排显示
    2. 如果父元素的宽度不能显示所有浮动元素，那么会从最后一个元开始往前贴靠
    3. 如果贴靠了前面所有浮动元素之后都不能显示，最终会贴靠到父元素的左边或者右边

    字围现象

    浮动元素不会挡住没有浮动元素中的 文字，没有浮动的文字会自动给浮动的元素让位置

    应用：图文排版，百度百科就是典型案例

    在浮动流中浮动的元素是 不可以 撑起父元素的高度

    清除浮动

    本质：为了解决父级元素因为子级浮动引起内部高度为 0 的问题

    方式：

    • clear 属性：选择器 { clear: 属性值; }

    • 额外标签法：在浮动元素末尾添加一个空标签，如：<div style="clear: both"></div>

    • 触发 BFC

    • after 伪元素

      .clearfix:after {
  content: ".";
  display: block;
  height: 0;
  clear: both;
  visibility: hidden;
}
/* 兼容 IE6/7 */
.clearfix {
  *zoom: 1;
}

    # BFC

    概念：块级格式化上下文，官方话语就不多说介绍了

    目的：形成一个完整独立的空间，让空间里的子元素不会影响到外面的布局

    设置 CSS 样式触发 BFC：

    • float 不为 none（就是要有浮动）
    • position 不为 relative 和 static
    • overflow 为 auto scroll 和 hidden
    • display 为 table-cell 或 inline-block

    # 定位

    定位相关概念：W3school 定位 (opens new window)

    静态定位

    指定：position: static;

    特点：所有元素的默认定位方式，默认标准流

    相对定位

    指定：position: relative;

    基点：以 自己的左上角为基点 移动（相对于自己来移动位置）

    特点

    1. 不脱标 ，盒子仍在 标准流 中
    2. 通过边偏移移动位置，但是 原来所占的位置，继续占有

    绝对定位

    指定：position: absolute;

    基点

    • 若父级 没有定位：以 浏览器当前屏幕 为准对齐
    • 若父级 有定位：将依据 最近的已经定位（绝对、固定或相对定位）的父元素（祖先） 进行定位

    1. 完全脱标，完全不占位置
    2. 如果文档可滚动，绝对定位元素会随着它滚动，因为元素最终会相对于正常流的某一部分定位

    固定定位

    指定：position: fixed;

    基点：以 浏览器窗口作为参照物

    特点

    • 固定定位的元素，跟父亲没有任何关系 只认浏览器
    • 固定定位：完全脱标，不占有位置，不随着滚动条滚动

    叠放次序

    概念：当对多个元素同时设置定位时，定位元素之间有可能会发生重叠

    指定：z-index: 2

    1. z-index 的默认值是 0，取值越大，定位元素在层叠元素中越居上
    2. 如果取值相同，则根据书写顺序，后来居上
    3. 后面数字一定不能加单位
    4. 只有相对定位，绝对定位和固定定位有此属性

    常用技巧

    子绝父相：子级是绝对定位的话，父级要用相对定位
