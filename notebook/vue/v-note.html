    # 四大标杆

    官方介绍：方法 (opens new window) | 计算属性 (opens new window) | 侦听器 (opens new window) | 过滤器 (opens new window)

    过滤器于 V3.X 版本中已 移除

    计算属性

    与方法区别不大，唯一就是多了个 响应式依赖缓存机制，有利性能的提升

    响应式依赖缓存机制： 因为计算属性更多的用于 data 中的数据逻辑处理，所以这个响应式依赖中的 依赖 指的就是 data 中的数据，而只要 data 中的数据不发生改变，也就不会重新计算，进而有利性能提升

    应用场景：适合对多个变量或者对象（即 依赖）进行处理后返回一个结果值

    侦听器

    官方介绍：API (opens new window)

    data 中某个数据绑定 对应的一个侦听器，data 中的这个数据发生改变的时候，自然就触发这个侦听器

    1. watch 中的属性，一定是 data 中已经存在的属性（侦听属性），其值就是属性对应的数据，也就是需要侦听的对应会变化的内容
    2. 侦听属性的值是一个对象的时候，需要指定 deep 属性对对象进行深度监听

    应用场景：

    1. 异步操作：比如 Ajax 异步请求数据，使用 watch 来响应数据的变化
    2. 开销较大的操作：耗时比较大的操作

    计算属性 VS 侦听器 VS 方法

    1. 计算属性相比方法多了个响应式依赖缓存机制
    2. 侦听器主要用于异步操作或者开销较大的操作，这个计算属性做不到
    3. 因为 data 中的属性对应一个侦听器，所以应该尽可能的使用计算属性而不是侦听器，从而避免侦听器的滥用

    # 组件

    官方介绍：组件基础 (opens new window) | 深入组件 (opens new window)

    基本概念：组件是带有名称的可复用 实例，和 根实例 具备相同的选项（属性和方法）

    设计理念：把一个页面拆分成多个组件，多个组件之间形成 树 结构，从而构成一个完整的页面

    组件特性：共用性和复用性，共用和复用的同时，组件内的数据是其独享的，不会互相影响

    定义 & 注册

    官方介绍：定义和注册 (opens new window)

    全局组件特点：只要定义了，处处可以使用，性能不高（因为会一直存在，占据一定的空间），但是使用起来简单方便

    局部组件特点：定义了，要注册之后才能使用，性能比较高，使用起来有些麻烦

    组件通信

    官方介绍：Props (opens new window)

    父子组件：父 -> 子

    1. 静态传参：直接以属性的形式向子组件传递参数（相当于写死）
    2. 动态传参：v - bind 属性绑定的形式传递参数（是一个动态的 变量 或 表达式）

    父子组件：子 -> 父

    子组件通过 $emit 向外触发 事件（原生事件或者自定义事件） 的方式进行参数的传递

    emits 属性：属性的值可以是一个对象或者数组，指定子组件向外触发的事件集合，便于维护

    官方介绍：emits (opens new window)

    父子组件：双向数据绑定

    官方介绍：基础 (opens new window) | 深入 (opens new window)

    父子组件：传递参数校验

    官方介绍：Props 验证 (opens new window)

    单向数据流

    概念：子组件可以使用父组件传递过来的数据，但是绝对不能 修改 传递过来的数据

    原因：组件的复用性和公用性，如果可以修改父组件的数据可能会导致复用组件的紊乱

    Non-prop 属性

    官方介绍：inheritAttrs (opens new window)

    当父组件传递给子组件值，但子组件不想接收值时，子组件的 DOM 上会自动继承该属性，这就是 Non-prop 属性

    应用场景：样式的传递

    跨组件通信

    主要方式：provide / inject

    官方介绍：跨组件通信 (opens new window)

    插槽

    官方介绍：插槽 (opens new window)

    父组件使用子组件或者组件使用公用组件的时候，对应的子组件或者公用组件展示其 对应特定的内容

    动态组件 & 异步组件
